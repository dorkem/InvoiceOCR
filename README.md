### 요구사항
1. 프로젝트 개요
Snowflake 환경 내에서 LLM(Cortex) 모델을 활용하여 송장 이미지로부터 데이터를 추출하고, 실무자의 검토 효율을 높이기 위한 자동화 파이프라인을 구축함.

2. 주요 프로세스 (Multi-stage Extraction)
데이터의 정확도를 극대화하기 위해 단일 추출이 아닌 2단계 프로세스를 거칩니다.

    1단계: 전체 텍스트 추출 (Raw Text Extraction)

    - 송장 이미지 내의 모든 텍스트 정보를 가공 없이 추출하여 ocr_result_data 필드에 저장.

    2단계: 품목별 핵심 데이터 구조화 (Refined Extraction)

    - 추출된 전체 텍스트를 바탕으로 철근, 레미콘 등 각 품목 특성에 맞는 필수 컬럼(차량번호, 품목명, 현장명 등)을 추출하여 JSON 형식의 result 필드로 정제.

3. 상세 기능 요구사항
    -  모델 비교 및 최적화: Snowflake 환경에서 제공되는 다양한 LLM 모델들을 테스트하여, 송장 인식률이 가장 높은 최적의 모델을 선정함.

    - 검토 대상 자동 분류 (Human-in-the-Loop): LLM이 추출한 결과값의 신뢰도를 판단하여, 실무자의 추가 확인이 필요한 건에 대해 '현장 확인 필요 여부' 컬럼을 생성하고 플래그(Flag)를 부여함.

    - 예외 케이스 처리: * 수기 작성된 글씨, 직인에 가려진 텍스트, 이미지가 잘린 경우 등을 고려하여 추출 로직을 설계함.

        - 품목 카테고리가 불분명한 경우 '기타'로 분류하여 처리함.

4. 기술 제약 및 환경
    - 입력 데이터: 송장 이미지 외의 추가 정보(메타데이터 등)는 제공되지 않으며, 오직 이미지 분석만으로 데이터를 추출함.

    - 개발 환경: Snowflake 내부 데이터 플랫폼을 활용 (Cortex LLM 기능 사용).

    - 출력 형식: 데이터 가독성을 위해 정렬된(Aligned) 형태의 JSON 포맷으로 최종 결과물을 생성함.
